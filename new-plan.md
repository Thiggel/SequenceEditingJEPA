 We are not currently asking “how do we solve ARC?”
  We are asking:

  > Can a LeWM-like compressed latent world model trained only on low-level grid edit dynamics discover useful abstractions such as objects, parts, locations, and relations, without
  > us giving it object slots/proposals?

  That is a much cleaner scientific question.

  Core Hypothesis
  The hypothesis is not:

  > JEPA learns to choose the right ARC action.

  The hypothesis is:

  > If future dynamics are generated by persistent latent structures, then a compressed multi-step JEPA is pressured to represent those structures.

  For example, if pixels belonging to the same hidden object tend to appear, move, recolor, disappear, or transform together over time, then the cheapest predictive latent may
  encode:

  object identity
  object location
  object shape
  object color
  object growth frontier
  relations between objects

  That is the question.

  Important Correction
  You are also right that random pixel edits may destroy the semantic signal.

  If the world is:

  at each step, random cell gets random color

  then there is no object-level causal structure to discover. The best compression is just statistical pixel bookkeeping. The model cannot invent stable semantic objects if the
  dynamics do not contain stable semantic objects.

  So the key is not “low-level actions” alone. The key is:

  low-level actions generated by hidden object/process structure

  The model only sees pixel edits, but the edit sequence has object-level regularities.

  That is exactly analogous to motor actions: a baby does not receive “object labels,” but the world has persistent objects whose pixels move/change coherently.

  Main Design Principle
  We need a synthetic world where:

  observations/actions are low-level
  generating causes are object-level
  object labels are hidden during training
  object labels are used only for probes

  So we should absolutely not train with explicit object slots/proposals. But we can use hidden object definitions to generate trajectories and evaluate probes.

  Should We Use Object-Ordered Trajectories?
  Yes, but only as one condition.

  If we always generate:

  build object A fully
  then build object B fully
  then build object C fully

  then we have injected temporal object grouping. The model might learn objects because the trajectory hands them to it.

  That is not bad. It is one condition. But it is weaker evidence.

  The stronger experiment is to compare trajectory regimes:

   Regime                         What It Tests
  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   object-blocked                 easiest case: object identity is temporally obvious
  ─────────────────────────────  ────────────────────────────────────────────────────────
   object-interleaved             objects are still coherent, but not built contiguously
  ─────────────────────────────  ────────────────────────────────────────────────────────
   frontier-growth                object grows locally from seed; more natural
  ─────────────────────────────  ────────────────────────────────────────────────────────
   random-within-object           object membership matters but local order is random
  ─────────────────────────────  ────────────────────────────────────────────────────────

  ─────────────────────────────  ────────────────────────────────────────────────────────
   noisy semantic                 objects can be overgrown, trimmed, recolored
  ─────────────────────────────  ────────────────────────────────────────────────────────
   pure random edits              negative control: no semantic structure

  Then ask:

  > Under which regimes does the CLS latent recover object structure?

  That is the actual answer.

  The Minimum Synthetic World
  Start with scenes made of 1-5 objects on a small grid, maybe 12x12 or 16x16.

  Objects should include:

  solid rectangles
  hollow rectangles
  lines
  L/T/cross shapes
  checkerboards
  stripes
  small multi-part objects
  duplicated objects
  same-shape different-color objects
  different-shape same-color objects
  touching/near-touching objects
  diagonal-contact objects

  Do not start with full ARC-AGI. Make a generator where we know the hidden factors.

  Each object has hidden metadata:

  object_id
  mask
  color or palette
  bbox
  centroid
  shape type
  parts
  relations: left_of, inside, touching, same_color, same_shape

  The model never sees this metadata. Probes see it later.

  Actions
  Keep actions low-level:

  paint(row, col, color)
  erase(row, col)
  recolor(row, col, color)

  Maybe later add:

  swap two cells
  move one colored cell

  But first use paint/erase/recolor only.

  Action embedding:

  row embedding + col embedding + color embedding + op embedding

  No object identity in the action.

  Trajectory Construction
  Generate a final solved scene from hidden objects. Then produce edit trajectories.

  There are two equivalent directions:

  empty -> solved
  solved -> empty

  For training dynamics, both are fine if actions specify paint/erase/recolor. I prefer generating solved scenes first, then sampling an order of cell edits, then reversing if
  needed. This gives precise control.

  For each scene, create multiple trajectories.

  Trajectory Type 1: Object-Blocked

  empty
  paint all cells of object A
  paint all cells of object B
  paint all cells of object C
  solved

  Within each object, order can be random or frontier-growth.

  This tests the easiest emergence case.

  Expected result:

  - CLS should encode “currently building object A.”
  - rollout should be stable.
  - probes should recover object count/bboxes.

  But this is the most biased condition.

  Trajectory Type 2: Frontier-Growth
  For each object:

  1. choose seed cell;
  2. repeatedly paint a neighboring unpainted cell from that object.

  object starts as seed
  object grows along connected frontier

  This is better than arbitrary object-blocked order because it creates spatial continuity. The model can infer shape growth.

  For checkerboards, use object mask membership but frontier can hop through 8-neighbor links.

  Trajectory Type 3: Random-Within-Object
  Object A is selected, but pixels inside A are painted in random order.

  This removes local-growth cues but keeps temporal object identity.

  If object probes still work, the model is not just tracking growing boundaries.

  Trajectory Type 4: Interleaved Objects
  At each step choose one active object, paint one of its remaining pixels.

  Examples:

  A, A, B, A, C, B, C, A, ...

  The object is not built in one block, but each object remains a persistent process.

  This is important. Real environments often interleave changes.

  Trajectory Type 5: Global Random Target Pixels
  All target pixels are painted in uniformly random order.

  This is a negative-ish control. The final image contains objects, but the dynamics do not expose object coherence.

  If probes work here too, then maybe static compression alone is enough. If probes fail here but work under object-coherent dynamics, that supports your temporal-abstraction thesis.

  Trajectory Type 6: Noisy Semantic
  This addresses your “can objects be trimmed/changed later?” concern.

  Examples:

  paint object too large
  erase excess cells
  recolor object
  repair missing hole
  move from wrong color to right color

  This teaches the model that colored pixels are not irreversible.

  But this should be structured noise, not random chaos. For example:

  overgrow object boundary by 1 cell
  then trim boundary

  That is semantically meaningful. It teaches editable object boundaries.

  Trajectory Type 7: Random Off-Manifold
  Small amount only.

  random board
  random paint/erase/recolor actions

  This teaches generic low-level dynamics and action sensitivity.

  But if too much of training is this, the model learns a paint simulator, not object abstraction.

  I would cap this at maybe 5-10% initially.

  The Key Mixture
  I would start with a controlled sweep, not one mixture.

  Run separate datasets:

  D0: pure global-random target-pixel order
  D1: object-blocked
  D2: frontier-growth
  D3: random-within-object
  D4: interleaved objects
  D5: noisy semantic
  D6: 80% semantic + 20% random counterfactual

  Then compare probes.

  This is much more informative than guessing the perfect mixture.

  What The Model Sees
  At each training sample:

  state grid x_t
  action a_t = low-level edit
  future states x_{t+1}, x_{t+2}, ..., x_{t+K}

  The model sees no object IDs, no masks, no task labels.

  Encoder:

  one-hot grid -> patch/cell embeddings -> bidirectional Transformer + CLS
  z_t = CLS

  Predictor:

  causal Transformer or MLP-transition
  input: z_t plus action embeddings
  output: predicted z_{t+1}

  Rollout:

  z_t
   -> predictor(z_t, a_t) = zhat_{t+1}
   -> predictor(zhat_{t+1}, a_{t+1}) = zhat_{t+2}
   -> ...

  Loss:

  sum_k MSE(zhat_{t+k}, stopgrad encoder(x_{t+k}))

  Plus anti-collapse regularizer:

  - LeWM-style Gaussian regularization,
  - or VICReg,
  - or SIGReg,
  - or EMA target encoder,
  - plus Delta-JEPA LDAD variants.

  Why Multi-Step Helps
  One-step prediction can be local:

  paint one cell -> update one cell

  Multi-step prediction asks:

  after 8 low-level edits, what latent state will I be in?

  If the next 8 edits mostly continue building the same object, the model benefits from representing the partially built object and its remaining structure.

  But again: only if the trajectory distribution contains that structure.

  Hierarchy
  Hierarchy should be added after flat rollout.

  Low level:

  predict 1 edit ahead

  Mid level:

  predict 4 edits ahead from action chunk

  High level:

  predict 16 edits ahead from compressed action chunk

  The high-level predictor receives:

  z_t + action_chunk_embedding(a_t ... a_{t+15})

  and predicts:

  z_{t+16}

  The action chunk encoder should be a small causal/bidirectional transformer over low-level actions, not just concat+MLP once chunk lengths grow.

  What we want to see:

  chunk embeddings cluster by macro-process:
  - build rectangle
  - fill checkerboard
  - recolor object
  - trim overgrowth
  - complete hole

  This would be strong evidence.

  CLS Only
  For this specific question, I agree: use CLS bottlenecks.

  Do not use grid tokens as the main latent, because they can preserve pixel identity too directly.

  Use:

  CLS d=32
  CLS d=64
  CLS d=128
  CLS d=256

  Maybe include grid-token latent only as a negative/upper baseline:

  If grid-token rollout works but CLS does not, compression is the hard part.

  But the main question should be CLS.

  Object slots are also premature. They bake in the answer.

  Model Suite
  Do not explode combinations yet.

  Phase 1:

  M0: CLS64, 1-step
  M1: CLS64, 4-step rollout
  M2: CLS64, 8-step rollout
  M3: CLS128, 4-step rollout
  M4: CLS128, 8-step rollout

  Phase 2:

  M5: CLS128, 8-step + LDAD
  M6: CLS128, 8-step + VICReg
  M7: CLS128, 8-step + SIGReg
  M8: CLS128, 8-step + EMA target

  Phase 3:

  H1: low 1-step + high 4-step
  H2: low 1-step + high 8-step
  H3: low 1-step + high 16-step
  H4: H2 + LDAD

  Keep it small until probes show signal.

  How To Probe
  This is the most important part.

  Probe 1: static reconstruction
  Train a small decoder from frozen z_t to grid.

  But reconstruction alone is not enough. A latent can reconstruct without semantic organization.

  Probe 2: object factor probes
  From frozen z_t, predict:

  number of objects
  object colors present
  object bboxes
  object centroids
  object area counts
  which object is currently being built
  fraction complete per object
  whether object has overgrowth
  whether object has holes/missing cells

  Use linear probe first, then small MLP. Linear-probe success is stronger.

  Probe 3: pixel-to-object membership
  For each cell, predict hidden object_id or object/background from z_t plus cell coordinate.

  This tests whether object identity is encoded globally.

  Probe 4: action delta probes
  Compute:

  delta = z_{t+1} - z_t

  Probe whether delta reveals:

  - action row/col/color;
  - whether edited pixel belongs to object A/B/C;
  - whether action continues current object;
  - whether action is overgrowth/trim/recolor.

  Delta-JEPA should help here.

  Probe 5: rollout semantic accuracy
  Do not decode only pixels. Decode object factors after rollout:

  z_hat_{t+8} -> object count, bboxes, completeness

  If pixel reconstruction degrades but object factors remain accurate, that is interesting and very JEPA-like.

  Probe 6: attention maps
  Attention is weak evidence, but useful qualitatively.

  Compare CLS attention to hidden object masks:

  - IoU with current object;
  - IoU with all incomplete objects;
  - attention entropy;
  - whether heads specialize.

  Probe 7: latent clustering
  Cluster states by hidden variables:

  - same object being built;
  - same shape type;
  - same macro-process;
  - same relation.

  Check whether nearest neighbors in latent space share semantics more than pixel Hamming alone predicts.

  Probe 8: hierarchy macro-action clustering
  For high-level action chunk embeddings, test whether chunks cluster by:

  - object ID;
  - operation type;
  - shape type;
  - correction type.

  This directly tests your hierarchy claim.

  The “Wrong Trajectory” Question
  I would stop calling them wrong trajectories.

  Use three categories:

  semantic trajectories: generated by hidden object processes
  counterfactual trajectories: valid edits from semantic states but not on the sampled process path
  random trajectories: no hidden object process

  Counterfactuals are useful if we want action sensitivity. But too many can make the world look less semantic.

  So measure it:

  0% counterfactual
  10% counterfactual
  25% counterfactual
  50% counterfactual

  If object probes improve with some counterfactuals but collapse at 50%, that tells us the semantic/random balance matters.

  Does The Model Learn A Prior Of Wrongness?
  Potentially, yes.

  If most training states lie on semantic construction manifolds, then compressed latent space may represent those manifolds well and random states poorly.

  Random/off-manifold states could have:

  - higher prediction error;
  - higher uncertainty if modeled;
  - worse latent regularity;
  - poor decoder reconstruction;
  - unusual latent norm/statistics.

  This is exactly a good probe:

  Can JEPA surprise/error distinguish semantic states from random states?

  But be careful: if random states are visually weird, this can be trivial. Make hard negatives:

  - plausible object states with one wrong outgrowth;
  - object with wrong color;
  - checkerboard with broken parity;
  - two objects accidentally connected.

  What Would Convince Me
  Strong evidence:

  1. 8-step CLS JEPA has better object probes than 1-step CLS JEPA.
  2. Frontier/interleaved semantic trajectories produce stronger object probes than global-random target-pixel trajectories.
  3. Hierarchical chunk latents cluster by macro-actions without macro labels.
  4. Delta/LDAD improves action sensitivity without destroying object abstraction.
  5. Semantic state surprise is lower than corrupted/random state surprise.
  6. Nearest neighbors in latent space share object/process semantics, not just pixel similarity.

  Weak evidence:

  - decoder reconstructs board.
  - one attention head visually highlights an object once.
  - rollout MSE improves but probes do not.
  - model works only on object-blocked trajectories.

  What Would Falsify The Idea
  The idea is in trouble if:

  multi-step rollout improves only pixel reconstruction, not object probes
  hierarchy improves k-step MSE but macro-action clusters do not appear
  CLS bottleneck loses all object information
  random target-pixel trajectories perform as well as semantic trajectories
  direct autoencoder features are as good as JEPA features

  The last one matters. We need autoencoder/MAE/reconstruction baselines. Otherwise we cannot claim JEPA did anything special.

• Yes, broadly this is a good plan. But I would sharpen one thing: we should not expect
  “progress” to emerge from the state alone unless the model also has either future
  actions, a task/intention variable, or a biased trajectory distribution.

  A JEPA world model answers:

  given state x_t and action(s) a_t...a_t+k, what will the future state be?

  It does not automatically answer:

  what should happen next?

  That second question requires a policy, goal, task context, or learned action prior.

  So your doubt is correct: if the model sees a partial object and no information about
  future actions or task family, it has no reason to believe the object will be filled,
  rotated, translated, etc. If the action sequence is provided, it can simulate it. If
  the trajectory distribution repeatedly contains “partial box -> filled box,” then it
  may learn that this is a natural process, but that is a learned prior from data, not
  pure dynamics.

  Build-Up Alone Is Not Enough
  Building objects from empty grids can teach object persistence and grouping, especially
  under bottleneck + rollout. But it mostly teaches:

  these pixels tend to be created together

  It does not strongly teach:

  objects can transform while preserving identity
  objects have orientation
  objects can be completed
  objects can be translated
  objects can be repaired

  So yes, we need additional trajectory families that start from already drawn objects
  and modify them.

  The first trajectory set should include both:

  construction trajectories:
  empty -> object/scene

  transformation trajectories:
  partial/full scene -> changed scene

  Good First Trajectory Types
  I would start with these five:

  1. Frontier Object Build

  empty -> object grows from seed

  Tests whether the model discovers object membership from coherent growth.

  2. Interleaved Multi-Object Build

  object A grows, object B grows, A grows, C grows...

  Tests whether it can track multiple latent objects without contiguous construction.

  3. Completion

  incomplete box/line/checkerboard -> complete object

  This is ARC-like and gives “progress toward closure” structure.

  4. Transform Identity

  object -> rotated/translated/reflected/recolored object

  This tests whether latent space preserves object identity across change.

  5. Noisy Repair

  object with overgrowth / missing cells / wrong colors -> repaired object

  This tests whether the model learns editable object boundaries and separates semantic
  corruption from arbitrary randomness.

  Those are more likely to produce useful signal than pure empty-to-solved build-up
  alone.

  Why These Help
  The key thing we want is pressure for the model to represent invariants.

  Build-up teaches:

  which pixels belong together

  Completion teaches:

  what structure is implied by partial evidence

  Rotation/translation teaches:

  same object, different pose/location

  Recoloring teaches:

  shape identity separate from color

  Repair teaches:

  valid object manifold vs corrupted object manifold

  That is much closer to ARC.
